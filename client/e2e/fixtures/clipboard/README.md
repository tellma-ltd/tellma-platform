# Clipboard fixtures

Test vectors for the tm-grid paste ladder (spec 0004 §9.3) and the copy
format contract (§9.2). The Playwright specs feed these payloads to the grid
through a synthetic `ClipboardEvent('paste')` (`e2e/support/clipboard.ts`),
so they run on every browser engine without touching the OS clipboard.

> **Provenance: AUTHORED, not recorded.** The `excel/` and `sheets/`
> payloads are authored to the documented clipboard formats of the real
> applications (shape, attributes, quoting rules) — replace with recorded
> payloads when a manual capture pass against real Excel / Google Sheets is
> done. The `tellma/` payloads are **generated from the grid's own
> serializer** (`tmSerializeHtmlTable`) by `generate-tellma.mts`, so they
> always carry the exact `data-tm-*` grammar the grid writes.

Regenerate the generated set (from `client/`):

```sh
pnpm exec tsx e2e/fixtures/clipboard/generate-tellma.mts
```

Line endings are load-bearing (`.gitattributes` pins `-text`): TSV rows end
CRLF — including the last — and in-cell line breaks are a bare LF inside a
quoted field.

## What each file pins

### excel/

| File | Contract pinned |
| --- | --- |
| `simple-2x2.txt` | Excel's `text/plain` flavor: plain TSV, CRLF after every row including the last. |
| `simple-2x2.html` | Excel's CF_HTML body shape: `<html>` with Office XML namespaces, `mso-*` styles, `<col>` widths, `x:num` cells, `<!--StartFragment-->`/`<!--EndFragment-->` markers. The grid must parse the display strings and ignore all of it. |
| `quoted-cells.txt` | Excel TSV quoting: cells containing tab / newline / quote are quoted with inner quotes doubled; in-cell breaks are bare LF inside the quotes. Round-trips byte-for-byte through paste → copy. |
| `headers-roundtrip.html` | The copy-with-headers → Excel → re-copy shape: the header row is now the FIRST DATA ROW (`<tr><td>` — no `<thead>`, no `data-tm-*`; Excel strips foreign attributes). Drives the §9.3 content heuristic: skipped only where the cells match the target columns' headers position-wise. First row = the grid-editable story headers `Description` / `Qty`. |
| `numbers-de.txt` | German-locale Excel TSV (`1.234,56`). Documents the limitation that plain TSV carries no locale metadata — an `en` grid cannot know these are German numbers; source-locale parsing needs the HTML metadata (see `tellma/numbers-de.html`). |

### sheets/

| File | Contract pinned |
| --- | --- |
| `simple-2x2.html` | Google Sheets' `text/html` flavor: `<meta charset>` + `<google-sheets-html-origin>` wrapper and proprietary `data-sheets-value` / `data-sheets-numberformat` attributes. The embedded `data-sheets-value` JSON deliberately DISAGREES with the display text (999/888/777 vs 7/8.25/9) — the grid must ignore the attributes (§9.3) and paste the display strings. |

### tellma/

All generated by `generate-tellma.mts` from `tmSerializeHtmlTable` — the
single source of truth for the `data-tm-grid` / `data-tm-v` /
`data-tm-rowid` grammar (§9.2). Values reference the `grid-editable` story
(columns `description`…`agentId`, agent directory ids 11–16, tenant `t1`).

| File | Contract pinned |
| --- | --- |
| `typed-2x3.html` | Same-tenant (`t1`) typed paste: 2×3 number cells whose first display string (`1,5`) disagrees with its raw value (`1.5`) — proves the typed fast path writes `data-tm-v` raw values without parsing (an `en` parse would yield 15). |
| `full-rows.html` | A full-row copy: per-row `data-tm-rowid`, every story column (incl. the key-less accessor column), typed raw values throughout — the entity cells carry raw ids that a same-tenant paste writes WITHOUT invoking the resolver. |
| `with-headers.html` | Copy-with-headers: `<thead>` + `headers:true` inside `data-tm-grid` — pasted back, the header row is skipped via the metadata flag (no heuristic involved). |
| `cross-tenant.html` | Tenant `t2` payload whose raw entity ids are WRONG for their labels (13/16 vs the labels' true ids 11/12): a paste must refuse the cross-tenant raw ids and re-resolve the labels (§9.4). |
| `numbers-de.html` | `locale:"de"` metadata + German display strings, no raw values: parsing must go through the source-locale hint (`ctx.sourceLocale`) — `1.234,56` → 1234.56, `2,5` → 2.5. |
